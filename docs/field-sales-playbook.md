## 1. 30초 오프닝 스크립트

> "사장님 안녕하세요, 바쁘신데 잠깐만요.  
> 저 한끼비서라는 앱 운영하고 있는데요.  
> 오늘 뭐 먹을지 고민하는 사람한테 메뉴 추천해주는 앱이에요.  
> 근처 가게를 찾을 때 **가입 매장이 1순위로** 뜨거든요.  
> 잠깐 보여드려도 될까요?"

→ 폰으로 **pitch.html** 보여주기

---

## 2. 핵심 설명 포인트 (pitch.html 보여주면서)

**"귀찮아" 버튼이 핵심입니다:**
- 배달앱은 '뭐 먹을지' 고민부터 시작
- 한끼비서는 앱이 이미 메뉴를 정해줌
- "귀찮아" = **이미 그 메뉴 먹기로 결정한 손님**이 근처 가게 찾는 것
- 이 때 가입 매장이 1순위

**경쟁 차별점:**
- 배달앱 → 수수료 10~15% + 광고비 수십만원
- 한끼비서 → 월 39,000원 정액, 첫 달 무료

---

## 3. 자주 나오는 objection 대응

| 사장님 반응 | 대응 |
|------------|------|
| "배달앱이랑 다른 게 뭐예요?" | "배달 아니에요. 앱이 메뉴 정해주고, 근처에서 직접 먹으러 오는 손님이에요. 수수료 없어요." |
| "사람들이 그 앱 써요?" | "지금 초기라서 가입 매장이 적을수록 노출이 더 잘 돼요. 지금 들어오시면 경쟁 없이 1순위에요." |
| "효과 있을지 모르겠어요" | "첫 달 무료라서 한 번 써보시고 결정하시면 돼요. 계약도 없고요." |
| "바빠서..." | "지금 바로 안 하셔도 되고요, QR 찍어두시면 나중에 하실 수 있어요." |
| "얼마예요?" | "월 39,000원인데 첫 달은 무료예요. 한 명만 와도 본전이에요." |

---

## 4. 가입 유도 단계

1. pitch.html로 설명 완료
2. **"지금 바로 신청하시면 오늘 안에 등록해드릴게요"**
3. QR or 링크로 merchant-landing.html 열어줌
4. 사장님 폰으로 직접 입력 or 대신 입력해드림
5. 제출 완료 → 성공 화면 확인
6. **"담당자가 오늘 중으로 연락드릴 거예요. 연락 가능한 시간이 언제세요?"** 메모

---

## 5. 운영 흐름 (제출 후)

```
sales_leads 테이블 insert
    ↓
운영자 Supabase에서 확인
    ↓
사장님 연락 → 메뉴 매핑 확인
    ↓
merchant_accounts 생성 + stores 등록
    ↓
store_menus 입력 (취급 메뉴 매핑)
    ↓
merchant_status = 'active' 변경
    ↓
앱 추천 DB 반영 완료 → 사장님 알림
```

---

## 6. 일일 영업 목표

- 방문: 20곳
- 설명 완료: 8곳
- 신청 완료: 2~3곳 (전환율 25~30% 목표)
- 후속 연락 예약: 5곳

현장에서 특이사항(사장님 요청, 재방문 필요 등)은 담당자 노션/슬랙에 별도 기록하세요.
